- clamp(min, val, max) : apparemment permet de fixer une valeur en unité variable, comme les rem, mais en bornant par des valeurs en pixels : ainsi, même avec des valeurs rem trop grandes ou triop petites, on respectera une certaine taille.
- @contaioner : container query : comme les media queries, permet de définit des propriétés conditionnelles, mais au lieu que ce soit en fonction de l'écran complet, c'est fonction du parent (des sélecteurs indiqués dans la container query)... attention à la lisibilité je pense...
